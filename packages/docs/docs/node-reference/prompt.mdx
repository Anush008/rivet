## Overview

The Prompt Node represents a single message in a prompt (from the User, Assistant, or other). Prompt Nodes can then be assembled into a message chain using the [Assemble Prompt Node](./assemble-prompt), which can in turn be fed into a [Chat Node](./chat).

In some contexts, [Text Nodes](./text) are interpreted as Prompt Nodes from the User.

## Editor Settings

| Setting                 | Description                                                                                                                         | Default Value | Notes |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------- | ------------- | ----- |
| Type                    | The chat message type (User, Assistant, System, or Function).   | User    | Can be set dynamically as input   |
| Name                    | The name of the User or Function. Usually this is safe to ignore. | (optional) | Usually this is safe to ignore. |
| Enable Function Call    | Enable OpenAI function calling. | Off |     |
| Compute Token Count     | Adds a Token Count output, that computes the number of tokens the message contains. | Off   | Count is based on the tokenizer for gpt-3.5-turbo and gpt-4. |
| Prompt Text             | The text of the message.      | (required)  | Text can contain template variables, marked by "{{" and "}}" |

### Example: Question Answering Prompts

To create a prompt for answering a question about the text of an article:

1. Add a Prompt Node to your graph.
2. Make the prompt text something like: "Here is the article text: {{articleText}}. Answer the question: {{question}}." The `articleText` and `question` template variables will create inputs for the prompt node.
3. Connect the inputs to [Text Nodes](./text) containing the article text and the question.
4. Connect the output to a [Chat Node](./chat)
